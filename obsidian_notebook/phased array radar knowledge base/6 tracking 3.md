## 你抓对的核心点

- **数据按 dwell 离散到达** ✓
- **每个 detection 只是 `(p, t)`,没有身份标签** ✓ —— 这正是 tracking 之所以非平凡的根源
- **tracker 的核心任务之一是给每个 detection 找身份** ✓ —— 这个子问题在文献里有专门的名字,叫 **data association(数据关联)**
- **用运动模型来推断身份** ✓ —— 更准确地说,运动模型给出"这条 track 下一时刻应该出现在哪",然后看哪个 detection 落在合理范围内,落在范围内的就是候选
- **track = 对一个目标轨迹的估计** ✓

你说"detections 可以理解成 a set of `(p_i, t_i)`,在 t_i 位置 p_i 有一个目标但不知道是谁",这个写法其实就是多目标追踪问题的标准形式化,你已经在用学界的语言思考了。

## 几个值得加进 mental model 的细节

**1. detection 不一定只有位置。** 你假设"一个 detection 就是一个空间坐标"——这是当前代码的简化,但真实系统的一个 detection 通常还带:Doppler 径向速度(从频移直接测出来,非常便宜也非常好用)、SNR / 幅度、RCS、测量协方差等等。Doppler 尤其重要,因为它直接给你速度的一个分量,能大幅消除身份歧义——两个空间相邻但径向速度差很多的 detection,你一眼就知道不是同一个目标。

**2. detection 集合里有"垃圾"。** 真实一个 dwell 出来的不是干净的"真目标位置集合",而是:

```
detections = {真目标位置} ∪ {虚警 false alarm} − {漏检 missed detection}
```

虚警是噪声 / 杂波被误判成目标,漏检是目标真的在但这次没探到(检测概率 P_d < 1)。所以 tracker 不只要判身份,还要判"这个 detection 是真的属于某个目标,还是噪声该丢掉"。这也解释了为什么需要 **tentative → confirmed** 这种机制:单点起航迹太容易被噪声骗,要连续命中几次再确认。

**3. 运动模型在数学上是"先验"。** 你说用运动模型来判别身份,直觉完全对。更技术性的说法:tracking 本质是一个**贝叶斯推断**问题:

- 隐藏变量:目标的真实状态序列(轨迹)
- 观测:detections
- 先验:运动模型(目标怎么动是合理的)
- 似然:测量模型(给定真状态,测量长什么样)

卡尔曼滤波是这个推断在线性高斯假设下的闭式解,α-β 滤波是它的稳态特例。理解了这一层,你之后看各种 tracker 变体就只是在换"先验是什么"和"似然是什么"。

**4. track 的生命周期是个状态机。** 代码里只有 tentative / confirmed 两个状态,真实系统常见的更细分:birth → tentative → confirmed → coasting(丢了几个 dwell 但还没放弃,纯靠预测往前推)→ deletion。Coasting 状态尤其重要,因为目标短暂被遮挡或低 SNR 漏检是常态,不能一漏就删。

## 一句话压缩你的 mental model

最简版本:

> **Tracking = 在不知道身份的位置测量序列上,用运动模型作为先验,反推每个目标的轨迹。**

更完整一点:

> **Tracking = 在含噪声、含虚警、有漏检的测量序列上,联合解决两个互相耦合的子问题——(1) data association:每个 detection 属于哪条 track(或者是虚警);(2) state estimation:给定关联结果,每条 track 的状态(位置/速度/…)是多少。**

这两个子问题互相依赖(状态估计影响关联,关联影响状态估计),所以各种 tracking 算法的本质区别,基本就是**怎么解耦或者怎么联合地解这两个问题**。GNN / JPDA / MHT / PMBM 之间的差异,核心都在这里。

你现在的 mental model 已经足够支撑你看懂大多数 tracking 文献的 introduction 部分了。
